In our model, we have the shortest distance for each residential property to its closest schools, hospitals, train stations etc. we will assume that easier access (smaller distance) to the closest facility will suggest a higher liveability for that criterion.

Data Downloading

Property Dataset
  • Web scraping from Oldlistings.com
  • API retrieving with Domain.com
External Dataset
  • Population density(SA2)
  • Income per person(SA2)
  • Crime cases(Postcode)
  • GDP (Nationwide)
  • Saving Rate (Nationwide)
  • Facility dataset(locations of schools, parks, hospitals etc.)

Data Preprocessing

  • Rental price regular expression
  • Residence type classification (Logistic Regression)
  • Found distances through OpenRoute Service API
  • Kept distance to the closest facilities (schools, parks etc.)

Modelling

Models we used for feature prediction are:
  • Linear Regression:
    crime case ~ postcode + year
    income per earner ~ SA2 + year
    population density ~ SA2 + year
  • Quadratic Regression: GDP ~ saving rate + year
Models we used for rental price prediction are:
  • Linear Regression
  • XGBoost
  • Random Forest

About us

We are Data Science students at The University of Melbourne. This website is built up in terms of statistical model illustrations. We hope that this website can bridge the communcation between the general population and our model and deliver some insights on rental price and liveability. Particularly, we hope that non-data-science background people can enjoy using and exploring this website. So that models constructed by us, the future data scientsts, are being applied and realised on a larger scale.
